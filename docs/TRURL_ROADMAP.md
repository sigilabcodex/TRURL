# TRURL Roadmap

## v0.1 — Repository + Markdown Editing

- Establish repository structure and authoring conventions.
- Store manuscript and canon in Markdown as primary data.
- Define initial schema and AI governance rules.

## v0.2 — AI Refactor Tools

- Add AI-assisted refactor scripts that produce diffs.
- Support branch-scoped AI editing workflows.
- Introduce initial continuity/canon validation checks.

## v0.3 — Web Editor

- Provide minimal web editor for manuscript and story-bible navigation.
- Add backend bridge for safe Git/GitHub operations.
- Preserve repository-first offline editing compatibility.

## v0.4 — Continuity Engine

- Build cross-document continuity indexing.
- Detect canon conflicts across chapters, character records, and timeline entries.
- Generate continuity reports for branch review.

## v0.5 — Collaboration

- Enable multi-author branch workflows.
- Add review-assist utilities for narrative merges.
- Improve conflict resolution guidance for overlapping edits.
